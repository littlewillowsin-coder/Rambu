NOTE: (Creates the Order & Items SIMULTANEOUSLY) When the buyer clicks "Place Order", backend code inserts data into both tables at the exact same millisecond in a single transaction: When the buyer clicks "Place Order", the system inserts data into the following two tables simultaneously within a single database transaction: 1️⃣ Table 1: 10_Table_orders (The Master Receipt) This table stores the high-level summary of the transaction. One row is inserted per checkout session. Fields populated: order_id: Auto-generated by the database (Auto Increment). buyer_id: The user_id of the logged-in buyer. order_date: The exact current timestamp (Auto-generated via DEFAULT NOW()). total_amount: The sum of all total_price values from the cart (calculated by your backend code before insertion). status_id: Set to 1 (which maps to "Pending" in the order_statuses table). shipping_address: The address the buyer entered or selected during checkout. order_notes: Any special instructions the buyer typed (can be NULL).
Orders Table
2️⃣ Table 2: 11_Table_order_items (The Line Items) This table stores the specific products being bought. One row is inserted for EVERY item in the cart. (e.g., If they buy 2 different harvests and 1 bag of fertilizer, 3 rows are inserted). Fields populated (for each item): order_item_id: Auto-generated by the database (Auto Increment). order_id: The order_id that was just generated in Table 1 (this links the item to the master receipt). seller_id: The user_id of the seller (Farmer, Nursery, or Supplier) who owns the listing. product_type: 'Harvest', 'Sapling', or 'Accessory' (tells the system which table the product lives in). product_id: The specific ID of the listing (e.g., the harvest_id or sapling_id). variety_id: Copied directly from the listing for quick display (set to NULL if the product is an Accessory). quantity: The exact amount the buyer is purchasing. unit_price: The price at the exact moment of purchase (copied from the listing so historical data isn't messed up if the seller changes their price later). total_price: Calculated as quantity × unit_price.
